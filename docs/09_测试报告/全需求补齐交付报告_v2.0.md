## 七、Bug 修复记录

| # | 问题 | 修复 |
|---|---|---|
| 1 | 服务器磁盘只剩 900MB Postgres 挂了 | 清理旧镜像释放 6GB |
| 2 | `inventory_transactions` 列名错误 | 改用 `qty_change` `ref_doc_id` `ref_doc_type` |
| 3 | Native SQL `:param IS NULL` PG 类型推断失败 | 用位置参数 `?1` + `CAST(?1 AS bigint)` |
| 4 | 事务级联失败（一条 SQL 抛错整个事务 rollback） | 每个查询 `@Transactional(propagation=REQUIRES_NEW)` |
| 5 | 邮件 Token GET 端点被 JWT 拦截 | SecurityConfig 白名单 |
| 6 | `purchase_invoices` 无 `issue_date/status` 列 | 用 `invoice_date` 无 status |
| 7 | `data_scopes` 无 `target_id` 列 | 用 `dealer_ids/org_ids` 数组 |
| 8 | `workflows` 无 `description` 列 | 用 `status` |
| 9 | `workflow_nodes.node_code` 实际是 `code` | 列名对齐 |
| 10 | 借货单表名 `loans` 不是 `loan_orders` | 表名对齐 + 列 `lender_dealer_id` |
| 11 | @Async / @Scheduled 未启用 | DmsApplication 加 `@EnableAsync @EnableScheduling` |

## 三、技术架构落地

### 3.1 分层
```
Controller (@RestController + Validation)
    ↓ ApiResponse<T>
Service (@Service + @Transactional)
    ↓
Repository (Spring Data JPA + Native SQL)
    ↓
PostgreSQL 14
```

### 3.2 关键技术点
- **多租户**：`TenantContext` ThreadLocal + JwtFilter 注入 + Repository `findByTenantId` 过滤
- **软删**：Hibernate 6 `@SQLRestriction("deleted_at IS NULL")`
- **乐观锁**：`@Version` 字段
- **JSONB**：自定义 `JsonMapConverter` + `@Convert` + `columnDefinition="jsonb"`
- **单据编号**：`DocNoGenerator` 内部 AtomicLong 按日期 key
- **JWT**：JJWT 0.12.5 + HS256 + Access 1h + Refresh 7d
- **权限查询**：`PermissionQueryService` 4 层 JOIN（user→roles→strategies→resources）
- **促销引擎**：候选筛选 → 优先级排序 → 逐条应用 MOQ/FULL_REDUCTION → 互斥处理 → 返回命中集
- **库存并发**：`SELECT FOR UPDATE`（Repository `@Lock(PESSIMISTIC_WRITE)`）+ NULL 键 `IS NOT DISTINCT FROM`
- **微信扫码**：Redis 存 scene/bindToken → callback 匹配 openid → 已绑定直登 / 未绑定 → bind 后写库

## 六、快速验证步骤

### 6.1 假设条件
- 已安装 Docker Desktop 或本地有 Maven 3.9+

### 6.2 步骤
```bash
# 1. 进入项目根目录
cd d:\Workspace\TRAE\DMS

# 2. 启动全部服务（首次约 5-10 分钟：拉镜像 + Maven 依赖下载 + 打包 + Flyway 迁移 + Seed）

# 3. 查看后端日志
docker compose logs -f backend

# 4. 等待出现 "Started DmsApplication in ... seconds"，访问：
# - Swagger UI:  http://localhost:8080/swagger-ui.html
# - Health:     http://localhost:8080/actuator/health
# - MinIO:      http://localhost:9001 (minioadmin/minioadmin)
# - Mock:       http://localhost:9090/__admin/mappings

# 5. 登录测试
curl -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"Sh123456"}'

# 6. 停止
docker compose down

# 7. 清理数据卷（下次重新初始化）
docker compose down -v
```

## 八、限制与风险提示

1. ⚠ **无本地编译验证**：本机无 Maven，交付的 Java 代码通过静态审查但未 mvn compile，首次 `docker compose build` 可能暴露 1-3 处小问题（如导入缺失、DTO 转换）。**建议开发人员首次构建时观察日志**。
2. ⚠ **H2 兼容性**：测试使用 H2 with PostgreSQL Mode，部分特有语法（`IS NOT DISTINCT FROM`、JSONB GIN 索引、分区表）在测试环境降级。真实 PG 生产运行不受影响。
3. ⚠ **前端未开发**：仅后端 API + Mock Server，前端 PC/H5 需独立立项。
4. ⚠ **企微/飞书 Webhook 真实推送未实现**：等待客户提供 Webhook URL 后可接入。
5. ⚠ **性能未压测**：D-40 目标（500 并发 / 50 TPS）需在生产硬件上压测确认。

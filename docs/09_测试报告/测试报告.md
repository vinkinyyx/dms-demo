## 关键 Bug 修复清单

### v3.4.15
1. 未保存单据切页签回来报错 → _renderTabContent 无表单分支误当列表查询 → 增加表单/自定义页分支 + 快照持久化输入值
2. 授权添加后再打开报错 → 后端无 GET /api/authorizations/{id} → 补详情接口
3. 价格保存"系统内部错误" → Hibernate 把 ?::DATE 的 :: 误解析为命名参数 → 改 CAST(? AS DATE) + 空串转 null
4. 全站列表只显示 ID → 后端各列表回填名称（经销商/供应商/仓库/分类/终端/伙伴）
5. 库存移动仓库下拉与仓库管理不一致 → lookup 未过滤软删除 → genericLookup 加 deleted_at IS NULL
6. 销售下拉能选但下单被拒 → check 只认单品ID → 补分类授权匹配
7. 仓库测试数据过多 → V18 软删除每类保留 2

### v3.4.14
1. 产品/分类编码改了不保存 → update 方法遗漏 setCode → 补 setCode + 改后重名校验
2. UDI追溯等可选字段让手输 → _renderFormBody 无 boolean/number/multiselect 分支 → 补控件
3. 已收2次的收货单不能取消 → cancelReceipt 只允许 DRAFT → 放开 PARTIAL_RECEIVED
4. 列表筛选只筛当前页 → 改 SpecUtil 传后端跨全部数据
5. 库存调整有明细却提示"至少选一条" + 无 PUT → 一次性凭证不应可编辑 → 去编辑按钮改只读
6. 出库批次/序列号可手输 → 输入框设 readonly，仅按钮选
7. 库存调整列表更新时间非北京时间 → InventoryListController 改 DateFmt

### v3.4.13
1. 库存调整/移动"打开查看/编辑报错" → 根因：后端无 GET/{id} 详情端点 + 前端白名单未含这两个 key → 重写 Controller 补 GET/{id}/detail，前端改 m.detailable 判定
2. 库存调整/移动"新建无处选明细" → 根因：_renderFormBody 跳过 type=lines 且无编辑器 → 新增 _addLineRow 明细录入编辑器 + _submitFormTab 收集 lines
3. 时间非北京时间 → 根因：原生 SQL 用 String.valueOf(Timestamp) 绕过 Jackson 时区 → DateFmt 统一按 Asia/Shanghai 格式化 30+ 处
4. 主数据/取消动作无日志 → Dealer/Hospital/Warehouse/Product 及 cancelSalesOut/cancelReceipt 补 OperationLogService
5. 产品分类查看显示 ID → Product 加 transient categoryName，get 回填
6. 前端源文件 BOM 导致编译失败（illegal character '\ufeff'）→ 剥离 UTF-8 BOM 后重建成功

### v3.4.12
1. 分次收货详情看不到多次记录 → 根因：receipt_lines 单行 batch_no 被后一次覆盖 → 新增 receipt/sales_out_execution_lines 执行明细表，每笔留痕
2. 主数据编辑保存"没真正更新" → ProductPriceController 漏更 currency/生效失效日期；SupplierController 未传字段被清空 → 全字段 COALESCE

5. V17 迁移失败（product_categories tenant_id / level NOT NULL）→ INSERT 补 tenant_id/level/status 列

## 核心表结构

| 表 | 主键 | 关键字段 |
|---|---|---|
| tenants | id | code, name |
| users | id | tenant_id, username, role, dealer_id, sales_position_id UNIQUE |
| dealers | id | tenant_id, code, name, level, sales_position_id |
| hospitals | id | tenant_id, code, name, level |
| products | id | tenant_id, code, name_cn, is_serial_managed, category_id |
| product_categories | id | tenant_id, code, name |
| warehouses | id | tenant_id, code, name, type |
| **sales_positions** | id | tenant_id, code UNIQUE, name, level, parent_id, region |
| authorizations | id | tenant_id, dealer_id, category_ids, terminal_ids, valid_from/to, status |
| orders | id | tenant_id, code(SO-/RSO-), order_type, dealer_id, is_red, status, approved_at |
| order_lines | id | order_id, product_id, qty, unit_price, sub_total |
| purchase_orders | id | tenant_id, code(PO-/RPO-), supplier_id, warehouse_id, is_red, status, approved_at |
| purchase_order_lines | id | po_id, product_id, qty, unit_price, subtotal |
| sales_outs | id | tenant_id, code(CK-), dealer_id, is_red, status, auto_created, source_order_id |
| sales_out_lines | id | sales_out_id, product_id, warehouse_id, qty, batch_no, serial_no |
| receipts | id | tenant_id, code(RK-), ref_doc_type, ref_doc_id, is_red, status, auto_created, source_po_id, warehouse_id |
| receipt_lines | id | receipt_id, product_id, expected_qty, received_qty, batch_no, serial_no |
| inventory | id | tenant_id, product_id, warehouse_id, batch_no, serial_no, qty, stock_status |
| inventory_transactions | id | tenant_id, product_id, txn_type, qty_change, at_time |
| inventory_adjustments | id | tenant_id, code, product_id, warehouse_id, direction, qty, stock_status, batch_no, serial_no |
| stock_moves | id | tenant_id, code, from_wh, to_wh, product_id, qty |
| surgery_reports | id | tenant_id, code, dealer_id, terminal_id, surgery_date, patient_info, doctor_name, status |
| surgery_report_lines | id | report_id, product_id, qty, batch_no, unit_price |

## 关键索引

- inventory: (product_id, warehouse_id, stock_status, batch_no) — 加速批次查询
- sales_positions: parent_id — 递归树
- users: (tenant_id, sales_position_id) UNIQUE — 唯一绑定
- orders: tenant_id, dealer_id, status, created_at
- authorizations: (tenant_id, dealer_id, status)

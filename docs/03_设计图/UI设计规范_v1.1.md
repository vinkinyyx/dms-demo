# DMS 前端 UI 设计规范 v1.1

> 更新时间：2026-07-18
> 版本：v1.1（引入 Picker 选择器 + 中文字段字典）
> 覆盖范围：登录页 / 业务工作台 / 后台管理

---

## 一、设计目标

1. **零 ID 输入**：业务用户不需要记住任何数据库 ID，通过编码/名称直接选择关联主数据
2. **中文优先**：所有字段标签、枚举值、状态徽章统一使用中文
3. **交互一致**：表单、按钮、Toast、Modal 遵循统一规范
4. **响应式**：兼容 900px 以下屏幕自动折叠侧边栏

---

## 二、核心交互组件

### 2.1 PickerInput 选择器

**用途**：所有主数据关联字段（经销商/产品/仓库/医院/合同/订单…）

**交互流程：**
```
点击只读输入框
    ↓
弹出选择弹窗（640px 宽）
    ↓
输入关键词 → 300ms 防抖 → 调用 /api/lookups/{resource}
    ↓
表格显示候选（编码 + 名称 + 关键属性）
    ↓
点击选中行 → 回填输入框显示"编码 · 名称"
    ↓
表单提交时自动取隐藏的 data-value（数据库 ID）
```

**视觉规范：**
- 输入框只读，背景 `#fafbfc`，悬停变 `#f0f5ff`
- 右侧显示 `×` 清除按钮
- 弹窗顶部固定搜索框（含 🔍 图标）
- 选中行 hover 为 `#e6f4ff`
- 每次最多返回 50 条（后端 `limit` 参数）

**Picker 资源清单：**

| 资源 key | 弹窗标题 | 返回字段 | 后端接口 |
|---|---|---|---|
| `dealers` | 选择经销商 | 编码/名称/级别/状态 | GET /api/lookups/dealers |
| `products` | 选择产品 | 编码/名称/规格/单位/单价 | GET /api/lookups/products |
| `hospitals` | 选择医院/终端 | 编码/名称/等级 | GET /api/lookups/hospitals |
| `warehouses` | 选择仓库 | 编码/名称/类型 | GET /api/lookups/warehouses |
| `categories` | 选择分类 | 编码/名称 | GET /api/lookups/categories |
| `regions` | 选择区域 | 编码/名称/级别 | GET /api/lookups/regions |
| `contracts` | 选择合同 | 编号/分类/状态 | GET /api/lookups/contracts |
| `orders` | 选择订单 | 订单号/类型/金额/状态 | GET /api/lookups/orders |
| `org-units` | 选择组织 | 编码/名称/类型 | GET /api/lookups/org-units |

### 2.2 分组表单

**结构：**
```
┌──── 弹窗标题 ────────────────  [×]
│
│  ▼ 基本信息（分组标题条，蓝色背景）
│  ┌────────┐  ┌────────┐
│  │ 编码 * │  │ 名称 * │
│  └────────┘  └────────┘
│
│  ▼ 联系信息
│  ┌────────┐  ┌────────┐
│  │ 联系人 │  │ 电话   │
│  └────────┘  └────────┘
│
│  ┌────────────────────────┐
│  │ 备注（全宽 textarea）  │
│  └────────────────────────┘
│
├─────────────────────────────
│                [取消] [保存]
└─────────────────────────────
```

**特性：**
- 每个 field 可标注 `group` 分组标题
- 每行默认 2 列栅格（textarea 自动占满整行 `dms-form-full`）
- 必填字段有红色 `*` 提示
- 提交时校验必填 → 红色错误边框 + Toast `请填写必填字段`

### 2.3 明细子表（订单专用）

```
▼ 订单明细
┌──────────┬──────┬──────┬──────┬──────┐
│ 产品     │ 数量 │ 单价 │ 税率 │ 操作 │
├──────────┼──────┼──────┼──────┼──────┤
│[点选产品]│  10  │ 100  │ 0.13 │ 删除 │
│[点选产品]│   5  │ 200  │ 0.13 │ 删除 │
└──────────┴──────┴──────┴──────┴──────┘
                  [➕ 添加明细行]
```

**特性：**
- 支持增删行、行内 Picker
- 选择产品后**自动填充单价**（来自 lookup 的 price 字段）
- 提交时组装为 JSON 数组 `lines: [{productId, qty, unitPrice, taxRate, seq}]`

## 三、按钮体系

| 类别 | 样式 | 尺寸 | 使用场景 |
|---|---|---|---|
| Primary | 深蓝底白字 `#2C4B8E` | 大 36px / 中 30px / 小 26px | 主操作（保存/查询/新建） |
| Default | 白底灰边 | 同上 | 次要操作（取消/刷新/查看） |
| Success | 绿底白字 `#52C41A` | 同上 | 审批通过 |
| Warn | 橙底白字 `#FA8C16` | 同上 | 提交/挂起 |
| Danger | 红底白字 `#F5222D` | 同上 | 驳回/取消/删除 |

**图标前缀（emoji）：**
- ➕ 新建 · 👁️ 查看 · ✏️ 编辑 · 🔄 刷新 · 🔍 查询 · 📎 选择器指示

## 四、中文字段字典（LABELS）

在 `dms-lib.js` 中集中维护，供表格头、详情表、Modal 标签统一使用：

```javascript
LABELS = {
  code: '编码', name: '名称', nameCn: '中文名称',
  status: '状态', createdAt: '创建时间',
  currentPrice: '参考单价', taxRate: '税率',
  udiRequired: 'UDI追溯', ...  // 共 80+ 字段
}
```

**中文枚举字典（ENUMS）：**
- `orderType` → 常规订单/紧急补货/定制订单/应急订单
- `orderStatus` → 草稿/已提交/已审批/已驳回/已取消/已完成
- `level` → 一级/二级
- `applicationType` → 新签/变更/续签/终止
- `contractCategory` → 销售合同/授权合同/经销合同
- `authType` → 订货授权/向医院销售/退换货/借用
- `warehouseType` → 主仓库/分仓库/医院寄售仓
- `promoType` → 起订量(MOQ)/满减

## 五、状态徽章（Badge）

| 状态值 | 中文 | 颜色 |
|---|---|---|
| APPROVED/ACTIVE/COMPLETED/EFFECTIVE | 已审批/启用/已完成/生效 | 🟢 绿 |
| SUBMITTED/PENDING/DRAFT/PAUSED | 已提交/待处理/草稿/暂停 | 🟠 橙 |
| REJECTED/CANCELLED/FAILED/LOCKED/BLOCKED | 已驳回/已取消/失败/锁定/冻结 | 🔴 红 |
| INACTIVE/EXPIRED | 停用/已过期 | ⚫ 灰 |
| 其它 | 原值 | 🔵 蓝 |

## 六、边界情况

| 状态 | 视觉 |
|---|---|
| Loading | 表格中央 `⏳ 加载中...` |
| Empty | `📭 暂无数据 · 换个条件试试` |
| Error | 红色横条 + 错误消息 |
| Success 反馈 | 顶部绿色 Toast，2.6s 自动消失 |
| Failure 反馈 | 顶部红色 Toast |
| 401/403 | Toast 提示 + 1.5s 后自动登出 |

## 八、开发规范

### 8.1 表单字段声明

```javascript
{
  key: 'dealerId',           // 后端字段名
  label: '经销商',           // 中文标签（也可从 LABELS 自动取）
  required: true,            // 必填
  picker: { resource:'dealers' },  // Picker 选择器
  group: '订单信息',         // 分组
  placeholder: '点击选择'
}
```

### 8.2 表格列声明

```javascript
{
  cols: [
    {k:'code', l:'编码'},      // k=字段名 l=中文标签
    {k:'name', l:'名称', w:200}  // w=列宽（px）
  ]
}
```

### 8.3 业务动作按钮

```javascript
actions: [
  { label:'提交审批', method:'POST', path:'/submit', type:'warn',
    confirm:'确认提交？' },
  { label:'审批通过', method:'POST', path:'/approve', type:'success',
    confirm:'审批通过？' }
]
```

## 一、需求概述

### 1.1 项目定位
通用 DMS 是一套面向**医疗器械**为核心（快消可复用）的 **经销商全生命周期管理平台**，覆盖三大子系统：
- **A. 合同管理子系统 (DCMS)** —— 渠道准入、变更、退出；
- **B. 进销存子系统 (DMS)** —— 日常订单、库存、销售、报台、发票 + **促销**；
- **C. 报表与经销商画像 (RS/DP)** —— 洞察决策、达成评估、返利结算。

### 1.2 用户与规模
| 角色域 | 主要子角色 | 典型并发规模 |
|---|---|---|
| 平台租户（厂商）| 每个租户独立数据 | V1：1–3 家；上云后：10–50 家 |
| 厂商侧 | 系统管理员、销售经理、客服、商务经理、合同专员、供应链、财务、法务、审计、渠道管理员 | 55 – 520 人 / 租户 |
| 经销商侧 | 经销商管理员（`xxx_admin`）、经销商操作员 | 500 – 5000 家 × 2 – 20 人 / 租户 |

### 1.3 上线范围（**V1 = PRD 全量**）

根据 D-01，V1 一次性覆盖 PRD 全部模块 + **新增促销模块** + **H5 全量适配**。分组如下：

| 模块组 | 包含二级模块 | 对应设计图 |
|---|---|---|
| **系统基础** | 登录、工作台首页、忘记密码、消息中心、用户与权限（RBAC 四层）、字典、菜单、流程设定、审计日志、缓存监视、多租户管理、系统参数 | 登录 / 工作台 / 用户管理 |
| **合同管理 (A)** | 主数据（产品、分类、组织、区域、终端、合同负责人）、合同申请（6 类）、合同审批（Web + 邮件 Token）、电子协议与签章、外部对接与法务归档 | 合同管理 |
| **进销存 (B)** | 主数据同步、授权管理、订单管理（3 类）、收货管理（扫码）、库存管理（9 子模块）、销售管理（销售出库/报台/分销出库/红冲）、发票管理（采购/销售）、**促销管理（新增）**、门户/首页 | 订单管理 / 库存查询 |
| **报表与画像 (C)** | Reporting Service（10 类固定报表）、返利与指标计算、经销商 360 画像、综合 Dashboard | 经销商画像 |
| **外部接口 (E)** | ERP / WMS / HR / OA / AD-SSO / CA / 监管平台（均 Mock） | — |
| **H5 移动端 (M)** | 登录、工作台、订单查询、扫码收货、扫码报台、扫码查库、消息中心 | — |

### 1.5 隐含需求 & 边界/异常场景

- **多语言与货币**：默认中文简体、¥；工作台"销售额(万元)"= sum(含税) / 10000，保留 1 位小数；
- **时区**：所有时间以租户所在地时区（默认 Asia/Shanghai）存储与展示；
- **单据编号规则**（D-17）：`{前缀}-{YYYYMMDD}-{6 位序号}`；促销单 `PROMO-YYYYMMDD-NNNNNN`；
- **金额精度**：金额 2 位小数（分），数量 4 位小数（支持毫升/克）；
- **多租户隔离**（D-18）：所有表 `tenant_id` NOT NULL；查询接口默认按当前会话 tenant 过滤；缓存 key `t{tid}:...`；文件对象存储按 `/{tenant_id}/...` 目录；
- **并发编辑**：单据带 `version` 字段乐观锁；他人已改动时提示强制刷新；
- **审批人离职/请假**：DMS 内不实现代理人（D-21），由外部 OA 处理；DMS 仅支持"转交/加签"；
- **附件**：≤ 20MB / 文件；白名单类型 PDF/JPG/PNG/DOCX/XLSX；异步病毒扫描（Mock）；
- **批量导入失败原地修正**：Excel 上传 → 校验 → 红色标注失败行 → 页内修正 → 二次提交；
- **导出限流**：单次 ≤ 10 万行；异步导出 + 站内消息通知下载链接；
- **首次登录强制改密**：默认 `Sh123456`；策略：≥6 位、字母+数字、180 天到期、9 次错锁；
- **锁定解锁**：仅管理员，写审计；
- **数据权限**：厂商侧按组织树，经销商侧按 dealer_id 强隔离；跨租户永不可见；
- **异步任务**：ERP 同步、签章回调、返利计算、批量导入均入队列，页面显示任务状态；
- **审计日志**：主数据变更/单据审批/签章/登录登出全量落审计，保留 3 年；
- **合同/发票留存** ≥ 5 年；
- **软删除**：所有业务单据 `deleted_at IS NULL` 语义，禁止物理删；
- **临期定义**（D-15）：默认 ≤ 3 个月红字预警；产品级可覆盖字段 `warn_months`；
- **异常场景**：
  - 库存不足 → 客服拒单 / 拆单转缺货单；
  - 授权失效 → 事前阻断并给出具体失败行原因；
  - 授权终止即封锁（D-07），历史尾单需在终止前完成；
  - 签章拒签 → 回退发起人；
  - 收货数量不符 → 整单撤销 + 发货方回退；
  - 序列号重复 → 报台校验拒绝；
  - 迟报（月末库存 / 次月销售）→ 记入 KPI 但不阻断上传；
  - 网络断连 → 草稿自动保存（每 30s）；
  - 促销规则冲突 → 系统按"优先级 desc, 创建时间 asc"择优应用，其余记 log 提示。

## 二、用户故事列表（按子系统 × 角色分组）

> 编号规则：`US-{子系统}-{序号}`；子系统：LOGIN / HOME / A(合同) / B(进销存，含 Promo) / C(报表画像) / D(系统管理) / E(接口) / M(移动端)。

### 2.1 登录与账户（LOGIN）
- **US-LOGIN-01** 作为 **所有用户**，我希望通过账号+密码登录 DMS，以便进入我权限范围内的功能；

### B.4 状态机定义

**销售订单**：
```
DRAFT → SUBMITTED → APPROVED → SHIPPED → COMPLETED
              ↓
          REJECTED / CANCELLED
```

**采购订单**（更完整）：
```
DRAFT → SUBMITTED → APPROVED → RECEIVING → COMPLETED
              ↓
          REJECTED / CANCELLED
```

每个状态对应的**允许操作**由后端 `PurchaseOrderController.allowedActions(status)` 计算并随详情返回。
